# 紫霄雷霆

一款竖版像素弹幕射击游戏（shoot 'em up）。三关、街机 1CC（一命通关）设计——硬核但公平，靠熟练度达成"不续关通关"的成就感。开局可在三架风格迥异的战机中择一出击。纯网页运行，支持 PC 与平板。

- **引擎**：Phaser 3 + TypeScript + Vite
- **音效**：全部由 Web Audio 在运行时**程序化合成**（音效 + 每关芯片音乐），无音频文件
- **美术**：初版为运行时**程序化像素美术**；正在升级为一套**文生图生成**的精修图集——资源索引见 [`art/ART_ASSETS.md`](art/ART_ASSETS.md) 与 [`art/assets.manifest.json`](art/assets.manifest.json)，替换时保持纹理 `TEX.*` key 不变
- **平台**：现代浏览器（桌面键鼠 / 平板触屏）

## 运行

```bash
npm install      # 安装依赖
npm run dev      # 启动开发服务器，浏览器打开 http://localhost:5173
```

构建发布版：

```bash
npm run build    # 类型检查 + 打包到 dist/
npm run preview  # 本地预览构建产物
```

`dist/` 是纯静态文件，可直接部署到任意静态托管（Netlify / Vercel / GitHub Pages / Nginx 等）。

## 操作

**PC（键盘）**
- 移动：方向键 或 `WASD`
- 精确模式（低速、显示判定点）：按住 `Shift`
- 炸弹：`X` 或 `空格`
- 暂停：`Esc` 或 `P`
- 开火为**自动**，无需按键
- 选机界面：`← →` 或 `A` `D` 切换战机，`Enter` / `空格` 确认，`Esc` 返回

**平板（触屏）**
- 拖拽屏幕任意位置移动飞机（相对拖拽，手指不会挡住机体）
- 右下 `BOMB` 按钮放炸弹，`FOCUS` 按钮切换精确模式，右上 `II` 暂停
- 选机界面：点选战机名页签切换，点「出击」确认

## 战机选择

开始游戏前（含练习模式选关后）进入**选择战机**界面，三架战机能力各异，所选机体会以雷达图直观对比并记忆到本地。命名取自《山海经》神兽神鸟。

| 战机 | 配色 / 级别 | 敏捷 | 射速 | 威力 | 防御（命数） | 弹药（炸弹） |
|------|------------|:---:|:---:|:---:|:---:|:---:|
| **青鸾** | 蓝 · 轻型 | 高 | 高 | 低 | 低（2 命） | 中（2） |
| **毕方** | 赤 · 中型 | 中 | 中 | 中 | 中（3 命） | 中（2） |
| **穷奇** | 金 · 重型 | 低 | 低 | 高 | 高（4 命） | 高（3） |

落实到数值：**敏捷**＝移速、**射速**＝开火间隔、**威力**＝子弹伤害倍率、**防御**＝起始命数、**弹药**＝起始炸弹。三机共用同一套火力强化（L1→L5）与弹幕，差异全在手感取向：青鸾以快制敌、穷奇以厚甲重炮稳推、毕方居中全能。配置集中在 [`src/ships.ts`](src/ships.ts)。

## 玩法要点

- **判定点极小**：真正的受击点远小于机体，子弹擦身而过是常态。
- **擦弹（Graze）**：贴近敌弹而不被击中可得分，鼓励高风险走位。
- **连击（Combo）**：连续击杀提升分数倍率，停手会衰减。
- **火力强化**：击落敌机掉落强化道具，火力 L1→L5 逐级变强；死亡掉一级。
- **炸弹**：清屏 + 短暂无敌 + 大量伤害，关键时刻保命。
- **命数 / 续关**：命数与起始炸弹取决于所选战机（青鸾 2 命 / 毕方 3 命 / 穷奇 4 命），固定 2 次续关。死亡原地复活并短暂无敌（非检查点回退）。
- **Clean Bonus**：未使用续关通关本关可获巨额奖励；全程不续关通关三关即达成 **★ 1CC CLEAR ★**。
- **练习模式**：从任意关卡开始、无限续关，用于熟悉弹幕。

## 关卡 / Boss

| 关卡 | 主题 | Boss |
|------|------|------|
| Stage 1 | ORBITAL APPROACH | CRIMSON WARDEN（三阶段） |
| Stage 2 | ION STORM | AZURE LEVIATHAN（三阶段） |
| Stage 3 | SOVEREIGN GATE | VIOLET SOVEREIGN（三阶段） |

每关约 2 分钟（约 90 秒杂兵波次 + 多阶段 Boss）。难度逐关递增（弹速、弹密度、敌机血量）。

## 调参

几乎所有手感数值都集中在 [`src/config.ts`](src/config.ts)：
- `PLAYER`：移速、判定/擦弹半径、火力间隔、无敌时长、死亡惩罚等
- `SCORING`：击杀/擦弹/连击/Clean Bonus 分值
- `BULLET`：子弹速度与对象池容量
- `DIFFICULTY`：三关的弹速 / 射速 / 血量倍率

关卡波次脚本在 [`src/levels/`](src/levels/)（`level1/2/3.ts`），Boss 在 `bosses.ts`，弹幕/移动模式在 [`src/systems/patterns.ts`](src/systems/patterns.ts)。

```
├── main.ts              Phaser 启动配置

art/                     美术资源索引（文生图重制）：ART_ASSETS.md + assets.manifest.json
                         （生成的图片交付到 art/incoming/，替换后保持 TEX key 不变）
```

## 设计取向

参考 CAVE / 东方等弹幕射击的"硬核但耐玩"哲学：第一次几乎不可能通关，但弹幕**可读、可躲**，配合擦弹、连击、炸弹与火力成长，让玩家在反复挑战中逐步掌握，最终体验"一命通关"的爽快。
